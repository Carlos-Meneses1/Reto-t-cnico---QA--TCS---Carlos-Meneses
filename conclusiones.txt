3. EJERCICIO 2 - PRUEBAS DE API
================================================================================

3.1 ENDPOINTS PROBADOS:
-----------------------
• POST https://api.demoblaze.com/signup
• POST https://api.demoblaze.com/login

3.2 TEST CASES:
---------------
Signup:
- TC-API-001: Crear nuevo usuario exitosamente
- TC-API-002: Validar rechazo de usuario duplicado
- TC-API-003: Validar campos requeridos

Login:
- TC-API-004: Login exitoso (retorna Auth_token)
- TC-API-005: Validar rechazo de usuario inexistente
- TC-API-006: Validar rechazo de password incorrecta
- TC-API-007: Validar estructura de respuesta (headers, content-type)
- TC-API-008: Verificar que passwords no se exponen en respuestas

3.3 PRINCIPALES HALLAZGOS:
--------------------------
• API siempre retorna status 200 (errores comunicados vía errorMessage)
• Mensajes de error específicos y consistentes
• Auth_token generado correctamente en login exitoso
• Passwords no aparecen en respuestas (buena práctica de seguridad)
• Sin rate limiting aparente (facilita pruebas, riesgo en producción)
• Generación de usuarios únicos con timestamp evita conflictos

6. DESAFÍOS Y SOLUCIONES
================================================================================

DESAFÍO 1: Tiempos de carga inconsistentes
Solución: cy.wait() estratégicos y validaciones con .should()

DESAFÍO 2: Manejo de alertas JavaScript
Solución: cy.on('window:alert') para captura y validación

DESAFÍO 3: API que siempre retorna status 200
Solución: Validación basada en campo errorMessage

DESAFÍO 4: Generación de usuarios únicos para tests
Solución: Uso de Date.now() en nombres de usuario
